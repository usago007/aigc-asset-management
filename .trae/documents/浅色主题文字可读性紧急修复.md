# 浅色主题文字可读性紧急修复计划

## 问题诊断

### 根因：浅色模式文字颜色太浅（对比度不足）

上一轮替换使用了过于保守的浅色值，导致大量文字在白色背景上难以阅读：

| 当前浅色值 | Hex值 | 问题 |
|-----------|-------|------|
| `text-gray-400` | #9ca3af | 几乎不可见！只能用于装饰性图标 |
| `text-gray-500` | #6b7280 | 仅适合描述性文字，不适合标签/数据 |
| `text-gray-600` | #4b5563 | 勉强可用，但作为正文还是偏浅 |

### WCAG 对比度标准要求
- 正文文字需要至少 4.5:1 对比度
- 大号文字需要至少 3:1 对比度
- 白色背景(#fff)上：
  - gray-500 (#6b7280) = **4.55:1** ✓ 勉强及格
  - gray-600 (#4b5563) = **5.79:1** ✓ 良好
  - gray-700 (#374151) = **8.97:1** ✓ 优秀
  - gray-400 (#9ca3af) = **2.85:1** ✗ 不合格！

## 实施策略

使用全局搜索替换 + Task agent 并行处理：

### Step 1: 修复全局CSS组件类（影响所有页面）
- `table-header`: `text-gray-500` → `text-gray-600 dark:text-gray-400`
- 其他已正确的不动

### Step 2: 批量替换高频错误模式（grep + replace）

**模式A: 字段标签** `text-gray-500 dark:text-gray-400` 用于标签场景 → `text-gray-700 dark:text-gray-400`
**模式B: 图标颜色** `text-gray-500 dark:text-gray-400` 用于图标 → `text-gray-600 dark:text-gray-400`
**模式C: 数据文字** `text-gray-600 dark:text-gray-300` 用于内容 → `text-gray-800 dark:text-gray-300`
**模式D: 残留无dark前缀** `text-gray-400` 无dark前缀 → 加上dark前缀或改深
**模式E: VideoGeneration/ImageGeneration section标题** `text-gray-100` → `text-gray-900 dark:text-gray-100`

### Step 3: 逐个修复特殊场景
- Sidebar 导航文字加深
- Header 面包屑
- shadcn/ui table/tabs/toaster
- TaskCard/ImageUploader

### 验收
- 切换浅色模式，检查每个页面：
  - 所有文字清晰可读
  - 标签和字段名明显可见
  - 图标不会消失
  - 表格数据易读
  - 按钮文字清晰
