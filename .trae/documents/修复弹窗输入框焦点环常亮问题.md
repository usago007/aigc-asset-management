# 修复弹窗输入框焦点环常亮问题

## 问题分析

从截图可以看到，"创建首图/尾图"弹窗中的"名称"输入框一直显示蓝色焦点环。

**根本原因**：Radix UI Dialog 打开时会自动 focus 第一个可聚焦元素，导致 Input 组件的 `focus-visible:ring-2` 样式被触发，焦点环一直显示。

**影响范围**：所有使用 Modal 组件的弹窗（KeyFrames、Shots、Assets、Brands、Customers、Projects、Briefs、Tasks、Reviews、Roles、Members、AIConfigPanel 导入/导出等）。

## 修复方案

### 方案选择

采用两个修复手段组合：

**1. 禁用 Dialog 自动聚焦** — 在 DialogContent 上添加 Radix 的 `onOpenAutoFocus` 阻止默认行为，防止弹窗打开时自动 focus 第一个元素。

**2. 优化 Input 焦点环样式** — 将 `focus-visible` 改为更精确的条件，仅在真正需要视觉反馈时才显示焦点环。

## 实施步骤

### 步骤1：修复 DialogContent 自动聚焦
- **文件**：`src/components/ui/dialog.tsx`
- **改动**：在 DialogContent 的 props 中禁用自动聚焦
```tsx
<DialogPrimitive.Content
  ref={ref}
  onOpenAutoFocus={(e) => e.preventDefault()}  // 新增
  className={cn(...)}
  {...props}
>
```

### 步骤2：优化 Modal 组件
- **文件**：`src/components/Modal.tsx`
- **改动**：在 Modal 的 DialogContent 上也禁用自动聚焦
```tsx
<DialogContent className={cn(width)} onOpenAutoFocus={(e) => e.preventDefault()}>
```

### 步骤3：验证
- 打开所有弹窗，确认输入框不会自动获得焦点环
- 手动点击输入框，确认焦点环正常显示
- TypeScript 编译通过
