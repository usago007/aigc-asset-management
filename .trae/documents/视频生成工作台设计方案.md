# 视频生成工作台设计方案

## 一、设计目标

基于即梦AI视频生成3.0系列接口（文生视频、图生视频-首帧、图生视频-首尾帧），为AIGC数字资产管理系统打造一个完整的视频生成工作台，覆盖从创意输入到视频产出的全流程。

## 二、支持的API能力

| 接口 | req_key | 功能 | 输入 | 输出 |
|------|---------|------|------|------|
| 文生视频 | `jimeng_t2v_v30_1080` | 文本→视频 | prompt | video_url |
| 图生视频-首帧 | `jimeng_i2v_first_v30_1080` | 首图→视频 | 1张图片+prompt | video_url |
| 图生视频-首尾帧 | `jimeng_i2v_first_tail_v30_1080` | 首图+尾图→视频 | 2张图片+prompt | video_url |

通用参数：
- `seed`：随机种子（-1为随机，相同seed+参数=相同结果）
- `frames`：121(5秒) 或 241(10秒)
- `aspect_ratio`：16:9, 4:3, 1:1, 3:4, 9:16, 21:9

## 三、工作台功能模块

### 模块1：快速生成（Quick Generation）

**适用场景**：快速测试Prompt效果、单镜头快速产出

**功能**：
- 选择生成模式（文生视频 / 首帧 / 首尾帧）
- 输入Prompt（带字数统计，建议≤400字）
- 上传图片（首帧/首尾帧模式需要，支持拖拽上传、本地预览）
- 参数配置：
  - 时长：5秒 / 10秒
  - 比例：16:9、4:3、1:1、3:4、9:16、21:9（首尾帧模式自动根据图片比例选择）
  - Seed：随机 / 自定义
- 一键提交生成任务
- 显示任务状态（排队中 → 生成中 → 已完成/失败）

### 模块2：批量生成（Batch Generation）

**适用场景**：多个镜头批量生成、同一Prompt多版本对比

**功能**：
- 批量导入镜头列表（CSV/Excel或手动添加）
- 每个镜头可独立配置Prompt和图片
- 批量提交任务，显示批量任务进度
- 支持QPS控制和并发限制
- 批量下载结果

### 模块3：版本对比（Version Comparison）

**适用场景**：同一镜头不同参数/seed的多版本选择

**功能**：
- 同一镜头多次生成结果展示
- 并排对比播放（最多4个视频同时播放）
- 标记"选用"版本（与现有GenerationVersion类型对接）
- 记录每次生成的完整参数（模型、seed、帧数、比例等）

### 模块4：任务管理（Task Queue）

**适用场景**：查看和管理所有历史任务

**功能**：
- 任务列表（状态筛选：排队中/生成中/已完成/失败/已过期）
- 任务详情：
  - 输入参数（prompt、图片预览、seed、frames等）
  - 输出结果（视频URL、状态、生成耗时）
  - request_id（用于排查错误）
- 过期提醒（视频URL有效期1小时，任务有效期12小时）
- 自动下载/转存到本地或对象存储
- 失败任务重试

## 五、数据结构设计

### 新增类型定义

```typescript
export interface VideoGenerationTask extends BaseEntity {
  // 任务标识
  taskId: string;              // API返回的task_id
  requestId: string;           // API返回的request_id
  
  // 生成配置
  mode: 'text-to-video' | 'image-to-video-first' | 'image-to-video-first-tail';
  reqKey: string;              // 服务标识
  prompt: string;              // 提示词
  
  // 图片输入
  firstFrameUrl?: string;      // 首帧图片URL
  firstFrameBase64?: string;   // 首帧图片base64
  lastFrameUrl?: string;       // 尾帧图片URL
  lastFrameBase64?: string;    // 尾帧图片base64
  
  // 参数
  seed: number;                // 随机种子
  frames: number;              // 帧数 121或241
  aspectRatio: string;         // 比例
  
  // 关联信息（可选）
  shotId?: UUID;               // 关联镜头ID
  projectId?: UUID;            // 关联项目ID
  
  // 状态
  status: TaskQueueStatus;
  progress?: number;           // 进度 0-100
  
  // 结果
  videoUrl?: string;           // 生成的视频URL
  videoExpiresAt?: string;     // 视频过期时间
  aigcMetaTagged: boolean;     // AIGC水印是否打标
  
  // 错误信息
  errorCode?: number;
  errorMessage?: string;
  
  // 耗时
  timeElapsed?: string;        // 链路耗时
  completedAt?: string;        // 完成时间
}

export type TaskQueueStatus = 
  | 'submitting'    // 提交中
  | 'in_queue'      // 排队中
  | 'generating'    // 生成中
  | 'done'          // 已完成
  | 'failed'        // 失败
  | 'expired'       // 已过期
  | 'not_found';    // 未找到
```

## 八、交互流程

### 核心流程：首尾帧生成

```
1. 用户选择"首尾帧图生视频"模式
2. 上传首帧图片 → 自动校验（格式、大小、比例） → 预览
3. 上传尾帧图片 → 自动校验 → 检查比例是否匹配 → 预览
4. 输入Prompt → 实时字数统计 → 超长提醒
5. 选择参数（时长/比例/Seed）
6. 点击提交 → 显示Loading → 调用submitVideoTask
7. 返回task_id → 创建轮询任务
8. 每5秒轮询 → 更新进度条 → 状态变化通知
9. 完成后 → 播放视频 → 可下载/关联镜头/加入版本对比
10. 视频URL过期前提醒下载（提前10分钟）
```

### 版本对比流程

```
1. 同一Shot的多次生成记录自动聚合
2. 进入"版本对比"视图 → 并排展示最多4个视频
3. 同时播放/暂停/进度同步
4. 点击"选用" → 标记为selected → 更新Shot的选用版本
5. 记录完整参数到GenerationVersion表
```

## 九、错误处理

| 错误码 | 含义 | 处理方式 |
|--------|------|---------|
| 50411 | 输入图片前审核未通过 | 提示用户更换图片，不可重试 |
| 50412/50413 | 文本审核未通过 | 提示修改Prompt，不可重试 |
| 50511/50516 | 输出审核未通过 | 提示可重试 |
| 50429/50430 | QPS/并发超限 | 自动延迟重试 |
| 50500/50501 | 内部错误 | 提示可重试 |
| not_found | 任务未找到/过期 | 提示任务已过期（12小时） |

## 十、Mock数据策略

由于暂不对接真实API，Mock数据应：

1. 模拟异步任务提交（setTimeout 1-2秒）
2. 模拟任务状态流转：in_queue(2s) → generating(5-15s) → done
3. 生成模拟视频URL（使用公开测试视频）
4. 模拟失败场景（随机或指定错误码）
5. 模拟视频URL过期（1小时后自动标记expired）
