# 找回丢失的仪表盘内容 - 修复计划

## 问题分析

仪表盘的4个子页面文件（Overview.tsx、Generation.tsx、DashboardAssets.tsx、DashboardTasks.tsx）**全部存在且代码完整**，没有被删除或修改。

**丢失原因**：在之前的 Token消耗追踪功能 实施过程中，sub-agent 在修改 App.tsx 添加路由时，**错误地覆盖了**之前已添加的 `/dashboard/generation`、`/dashboard/assets`、`/dashboard/tasks` 三个路由，同时侧边栏也从展开子菜单变回了单一菜单项。

## 修复步骤

### 步骤1：恢复App.tsx路由

在 `src/App.tsx` 中，将仪表盘路由从：
```tsx
<Route path="dashboard" element={<Dashboard />} />
<Route path="dashboard/overview" element={<Overview />} />
```

恢复为：
```tsx
import Generation from './pages/dashboard/Generation'
import DashboardAssets from './pages/dashboard/DashboardAssets'
import DashboardTasks from './pages/dashboard/DashboardTasks'

// 在Routes中：
<Route path="dashboard" element={<Dashboard />} />
<Route path="dashboard/overview" element={<Overview />} />
<Route path="dashboard/generation" element={<Generation />} />
<Route path="dashboard/assets" element={<DashboardAssets />} />
<Route path="dashboard/tasks" element={<DashboardTasks />} />
```

### 步骤2：恢复Sidebar.tsx子菜单

在 `src/components/Sidebar.tsx` 中，将仪表盘菜单从：
```tsx
{ path: '/dashboard', icon: <LayoutDashboard size={20} />, label: '仪表盘' },
```

恢复为：
```tsx
{
  path: '/dashboard',
  icon: <LayoutDashboard size={20} />,
  label: '仪表盘',
  children: [
    { path: '/dashboard/overview', label: '总览' },
    { path: '/dashboard/generation', label: '生成' },
    { path: '/dashboard/assets', label: '资产' },
    { path: '/dashboard/tasks', label: '任务' },
  ]
},
```

同时更新 `isActive` 函数：
```tsx
const isActive = (path: string) => {
  if (path === '/dashboard') return currentPath.startsWith('/dashboard')
  return currentPath.startsWith(path)
}
```

### 步骤3：验证

运行 `npm run build` 确认编译通过，确认所有路由和导航正常。
