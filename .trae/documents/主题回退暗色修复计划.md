# 主题回退暗色问题紧急修复

## 问题根因

**ThemeContext.tsx 第15行默认值为 `'dark'`**：

```tsx
const [theme, setTheme] = useState<Theme>(() => {
  const saved = localStorage.getItem('theme')
  return (saved as Theme) || 'dark'  // ← 问题在这里！默认 dark
})
```

当以下任一情况发生时，页面会回退到深色：
1. 用户首次访问（localStorage 无 theme key）
2. localStorage 被清除
3. 用户在 Settings 切换到浅色后，保存逻辑与 ThemeContext 不同步
4. 浏览器刷新后读取不到正确的值

## 修复方案

### 修改1: ThemeContext.tsx - 默认值改为跟随系统偏好 + 回退 light

```tsx
const [theme, setTheme] = useState<Theme>(() => {
  const saved = localStorage.getItem('theme')
  if (saved === 'light' || saved === 'dark') {
    return saved as Theme
  }
  // 回退：优先使用系统偏好，否则默认 light
  if (window.matchMedia('(prefers-color-scheme: dark)').matches) {
    return 'dark'
  }
  return 'light'
})
```

### 修改2: index.html - 添加防闪烁 script（可选但推荐）

在 `<head>` 中添加内联脚本，在 React 渲染前就设置好 class：

```html
<script>
  (function() {
    const theme = localStorage.getItem('theme') || 
      (window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light');
    if (theme === 'dark') {
      document.documentElement.classList.add('dark');
    }
  })();
</script>
```

### 修改3: 确保 Settings.tsx 的主题切换正确持久化

检查 `setTheme` 是否正确写入 localStorage（已在 ThemeContext useEffect 中处理，应无问题）
